# GPT와의 대화 흐름

## Phase의 제안

## 제안의 알기 쉬운 요약 : 다음 페이즈 제안 -> 복붙하기 쉽게 만들 파일 안내 -> 코덱스에게 보낼 문장 안내 -> 만들 파일 git에 추가하는 명령어 안내

1. AI: 이번 Phase에서 만들 세부 계획만 설명
    - 목표
    - 왜 지금 이 Phase인지
    - 무엇을 삭제/유지/변경할지
    - 리스크
    - 장기적인 면에서 안전하고 효율적인지
    - 다음 PR에서 확인할 기준

2. 사용자: 진행하자

3. AI: 담백하게 실행용 안내만 제공
    - 만들 문서 파일명
    - 문서 내용
    - git add / commit / push
    - Codex 주문서

4. 사용자: PR 확인해줘

5. AI: PR 리뷰
    - 코드 확인
    - 문제점 확인
    - 장기적인 면에서 안전하고 효율적인지 판단
    - 머지 판단
    - 판단 ok일 경우 머지 후 로컬 체크
    - 포맷 변경 시 commit/push
    - 안드로이드 스튜디오를 통한 수동 확인 리스트 안내
    - 판단 no일 경우 6-2 진행

6-1. 사용자: 문제 발생

6-2. AI: 문제 해결
    - 간단하게 끝난다면 사용자선에서 해결, 아니라면 codex한테 새 pr로서 주문
    - 당장 편하게 해결하는게 아니라 근본적으로, 장기적인 면을 고려해서 해결

7-1. 사용자: 정상 확인

7-2. AI: 다음 Phase(1번) 진행
