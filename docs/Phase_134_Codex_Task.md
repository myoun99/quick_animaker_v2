## Key rule

Avoid this mistake:

```dart id="w0wwfj"
class TimelineFrameRowsScrollBody extends StatelessWidget {
  const TimelineFrameRowsScrollBody({
    super.key = const ValueKey('timeline-frame-rows-scroll-body'),
  });

  @override
  Widget build(BuildContext context) {
    return KeyedSubtree(
      key: const ValueKey('timeline-frame-rows-scroll-body'),
      child: ...
    );
  }
}
```

That duplicates the same key.

The stable key must exist exactly once.

A safer pattern is:

```dart id="tnoopy"
class TimelineFrameRowsScrollBody extends StatelessWidget {
  const TimelineFrameRowsScrollBody({
    super.key,
    ...
  });

  @override
  Widget build(BuildContext context) {
    return KeyedSubtree(
      key: const ValueKey<String>('timeline-frame-rows-scroll-body'),
      child: ...
    );
  }
}
```

or attach the key only at the call site, but not both.
